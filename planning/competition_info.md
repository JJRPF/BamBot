# AI Agents: Intensive Vibe Coding Capstone Project (Kaggle)

## Overview
The AI Agents: Intensive Vibe Coding Capstone Project invites participants to build an AI agent using the concepts, tools, and best practices learned in Kaggle’s 5-Day AI Agents: Intensive Vibe Coding Course with Google. Participants will create a practical, real-world solution that demonstrates how AI agents can automate tasks, assist users, solve business challenges, or improve everyday workflows.

Submissions should showcase both the project's value and technical implementation through a Kaggle Writeup, public codebase, video demonstration, and project link.

---

## Timeline
- **Announced**: June 19, 2026
- **Submissions due**: July 6, 2026 at 11:59 PM PT

---

## Tracks and Awards
Judges will review submissions in four tracks. Winners receive Kaggle swag (12 swag packages total, 3 awards per track, non-monetary, does not award Kaggle points or medals):

1. **Agents for Good**
   - Submissions that help solve problems for humanity.
   - *Examples*: Optimizing agriculture, managing public health, advancing education, or supporting art and literature.
2. **Agents for Business**
   - Agents designed to solve compelling business problems with cost or revenue on the line.
   - *Examples*: Managing expense submissions, highlighting pipeline actions, driving insights, or creating new products.
3. **Concierge Agents**
   - Personal AI agents to streamline and simplify people’s lives while keeping personal information safe and secure.
   - *Examples*: Managing invite lists for a party, planning a garden, or helping manage complicated medications.
4. **Freestyle**
   - Innovative agent projects that don't fit into other tracks but demonstrate best practices of agent development and deployment.
   - *Examples*: Keeping track of concert recordings, decoding cursive for historians, or tracking satellite positions.

## Submission Requirements
A valid submission must contain:
1. **Kaggle Writeup**
   - Served as the project report (max 2,500 words).
   - Must select a Track to submit.
   - Must be submitted prior to the July 6 deadline.
2. **Media Gallery**
   - Attached to the Writeup.
   - Must include a cover image and the required video.
3. **Video**
   - Attached to the Media Gallery.
   - Must be **5 minutes or less** and published on YouTube.
4. **Public Project Link**
   - URL to a working product/interactive demo (publicly accessible, no login/paywall), OR
   - URL to a public code repository (e.g. GitHub) with detailed setup instructions if a live demo is not feasible.

## Key Course Concepts to Apply
Submissions must demonstrate what you've learned by applying at least **three (3)** of the following key concepts:
- **Agent / Multi-agent system (ADK)** (Demonstrated in: Code)
- **MCP Server** (Demonstrated in: Code)
- **Antigravity** (Demonstrated in: Video)
- **Security features** (Demonstrated in: Code or Video)
- **Deployability** (Demonstrated in: Video)
- **Agent skills (e.g., Agents CLI)** (Demonstrated in: Code or Video)

## Evaluation Rubric

### Category 1: The Pitch - Problem, Solution, Value (30 points total)
- **Core Concept & Value (10 points)**: Central idea, track relevance, innovation, value, and clear role of agents.
- **YouTube Video Submission (10 points)**: Clarity, conciseness, and quality within the 5-minute limit (covering Problem Statement, why agents, Architecture, Demo, and Build details).
- **Writeup (10 points)**: Articulation of the problem, solution, architecture, and project journey.

### Category 2: The Implementation - Architecture, Code (70 points total)
- **Technical Implementation (50 points)**: Architecture quality, code quality, meaningful agent use, clever tool usage (existing toolsets), and helpful comments. *Do not include API keys or passwords.*
- **Documentation (20 points)**: A `README.md` file explaining the problem, solution, architecture, setup/install instructions, and relevant diagrams/images.
